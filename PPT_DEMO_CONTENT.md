# CounselMan + MediPlat 시연용 PPT 구성안

## 0. 사용 방법
- 이 문서는 PPT 슬라이드 원고입니다.
- 각 슬라이드는 `넣을 항목`을 그대로 박스/불릿으로 배치하고, `설명`은 발표 대본으로 사용하면 됩니다.
- `데모 화면/URL`은 실제 시연 순서에 맞춰 캡처 또는 라이브 데모에 사용하세요.

---

## Slide 1. 프로젝트 소개
### 넣을 항목
- 프로젝트명: CounselMan + MediPlat
- 한 줄 소개: 상담 업무를 접수부터 기록, 문자 발송, 통계, 운영관리까지 통합한 플랫폼
- 구성: `csm`(업무 시스템) + `mediplat`(통합 플랫폼)

### 설명
- 본 프로젝트는 병원/기관 상담 업무를 표준화하고, 상담 데이터의 누락을 줄이며, 문자 커뮤니케이션과 운영 관리를 한 화면 체계로 연결하는 것이 목표입니다.
- `MediPlat` 로그인 후 `CounselMan`으로 연동되는 구조로, 기관별 운영과 권한 관리를 일관되게 지원합니다.

### 데모 화면/URL
- `https://dev.sosyge.net/login` (MediPlat)

---

## Slide 2. 해결하려는 문제
### 넣을 항목
- 상담 접수/기록/후속 연락이 분절됨
- 담당자별 기록 품질 편차
- 문자 발송 이력/비용 추적 어려움
- 기관별 운영 기능(권한, 템플릿, 공지, 기능 on/off) 관리 부담

### 설명
- 기존에는 접수, 상담기록, 문자 발송, 통계가 도구별로 분리되어 업무 재입력과 누락 가능성이 높았습니다.
- 본 시스템은 데이터 흐름을 하나로 연결해 운영 리스크를 줄이고, 관리자가 정책을 중앙에서 통제할 수 있도록 설계했습니다.

## Slide 3. 사용자/권한 구조
### 넣을 항목
- 코어 관리자(Core): 기관/사용자/공지/전역 설정 관리
- 기관 관리자/실무자: 상담 접수, 상담 기록, 문자 발송, 공지 확인
- 권한 기반 메뉴 노출(예: 통계/병실현황판)

### 설명
- 권한에 따라 접근 가능한 메뉴와 기능이 분리됩니다.
- 이 구조는 민감 기능 오남용을 방지하고, 역할별 UI 복잡도를 낮춰 실제 업무 속도를 높입니다.

---

## Slide 4. 시스템 구성 및 기술 스택
### 넣을 항목
- Backend: Spring Boot 3.4.x, Java 17
- Template/UI: Thymeleaf, 정적 JS/CSS
- Data: MySQL, MyBatis + JPA 혼합
- Security: Spring Security
- 배포 형태: `csm` WAR, `mediplat` JAR

### 설명
- 서버 렌더링 기반 구조로 화면 응답 안정성과 운영 단순성을 확보했습니다.
- 데이터 접근은 MyBatis와 JPA를 함께 사용해, 기존 SQL 자산 활용성과 확장성을 동시에 가져갑니다.

---

## Slide 5. 핵심 기능 맵
### 넣을 항목
- 상담 영역: 상담 접수, 입원상담, 상담리스트, 공지사항
- 운영 영역: 상담일지관리(템플릿), 문자관리(상용구/서명/전송내역/요금)
- 분석 영역: 상담통계, 엑셀 추출
- 확장 영역: 병실현황판, 파일/음성 업로드 및 요약 흐름

### 설명
- 실무 흐름 기준으로 기능을 묶어, 사용자 입장에서 “입력 → 커뮤니케이션 → 분석”이 자연스럽게 이어지도록 구성했습니다.

## Slide 6. 시연 시나리오(End-to-End)
### 넣을 항목
1. MediPlat 로그인
2. CounselMan 진입(연동)
3. 상담 접수 등록
4. 입원상담 작성 및 템플릿 반영
5. 문자 발송(상용구/서명 선택)
6. 전송내역/요금 확인
7. 통계 확인

### 설명
- 이번 시연은 실제 업무 하루 흐름을 1개의 시나리오로 압축해 보여줍니다.
- 기능 단위 데모가 아니라 “데이터가 어떻게 이어지는지”를 중심으로 설명하면 설득력이 높습니다.

## Slide 7. 상담 업무 화면(접수/리스트/상세)
### 넣을 항목
- 접수 생성 및 상태 변경
- 리스트 필터(기간/상태/유입경로/키워드)
- 상세 상담 이력 확인

### 설명
- 접수 단계에서 표준 항목을 확보하고, 리스트에서 즉시 필터링해 업무 우선순위를 정할 수 있습니다.
- 실무자는 빠른 조회, 관리자는 진행 현황 점검이 가능합니다.

### 데모 화면/URL
- `https://dev.sosyge.net/csm/counsel/reservation?status=RESERVED`
- `https://dev.sosyge.net/csm/counsel/list?page=1&perPageNum=10`

## Slide 8. 상담 품질 강화(템플릿/첨부/요약)
### 넣을 항목
- 상담일지 템플릿 기반 입력
- 파일/음성 업로드 및 이력 관리
- 요약/추출 기능으로 문서화 효율 개선

### 설명
- 템플릿으로 기록 품질을 표준화하고, 파일/음성 기반 자료를 상담 건과 연결해 추적성을 높였습니다.
- 상담자가 핵심 판단에 집중하도록 반복 문서 작업 부담을 줄이는 방향입니다.

## Slide 9. 문자관리(상용구/서명/전송)
### 넣을 항목
- 상용구관리: 자주 쓰는 안내 문구 관리
- 서명관리: 발신 서명 표준화
- 전송내역/요금관리: 발송 결과와 비용 가시화

### 설명
- 문자 발송의 품질과 일관성을 높이기 위해 상용구와 서명을 분리 관리합니다.
- 이력/요금 화면을 통해 운영자는 커뮤니케이션 성과와 비용을 함께 모니터링할 수 있습니다.

### 데모 화면/URL
- `https://dev.sosyge.net/csm/smsSetting?page=1&perPageNum=10`
- `https://dev.sosyge.net/csm/cardsetting?page=1&perPageNum=10`
- `https://dev.sosyge.net/csm/smslog?page=1&perPageNum=10`
- `https://dev.sosyge.net/csm/rate`

- 단순 UI 변경이 아니라, 실제 운영 이슈(오입력/중복저장/닫기 동선 혼선)를 줄이는 안정화 작업입니다.

### 넣을 항목
- MediPlat에서 서비스 선택 후 CounselMan 연동
- SSO shared secret 기반 연계
- 환경별 서비스 엔드포인트 관리

### 설명
- 서비스 허브(MediPlat)와 업무 앱(CounselMan)을 분리해 확장 가능성을 확보했습니다.
- 운영 환경(LOCAL/DEV/PROD)별 설정 분리로 배포/운영 실수를 줄일 수 있습니다.
