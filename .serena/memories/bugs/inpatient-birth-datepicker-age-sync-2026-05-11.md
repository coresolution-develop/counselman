Symptom: On design/inpatient-consultation.html, manual birth date typing populated age, but choosing a date from the custom MPDatePicker did not reliably populate age.
Root cause: The birth field logic was split between manual input formatting and generic change handling, while the custom datepicker only emitted generic input/change events. The datepicker selection path needed an explicit event that carries the selected value and routes through the same birth sync/calculation method.
Fix: Added an `mp-datepicker-change` CustomEvent from assets/js/datepicker.js after datepicker selection, and changed the inpatient birth field to use `x-model="patient.birth"` with a single `syncBirth(event)` handler for input, change, and the custom datepicker event. `syncBirth` normalizes YYYY-MM-DD text, updates `patient.birth`, and always calls `calcAge()` so incomplete/invalid values clear age through existing validation.
Verification: `node --check src/main/resources/static/assets/js/datepicker.js` passed. Inline script parsing for design/inpatient-consultation.html passed via Node Function parsing. Similar pattern scan found no other data-mp-dp-editable birth/age field in design templates.
